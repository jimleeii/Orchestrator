Markdown.Policy Skill

Contains writing profile, style, and policy for orchestrator-generated markdown.

Guidelines:

- Tone: concise, professional, active voice.
- Heading usage: structure content for skimmability — include a short TL;DR at the top for longer pages.
- Examples: Inline code fences for commands; use bullet lists for checks and short enumerations.
- File linking: prefer workspace-relative links, and avoid absolute `file://` paths.

Include a short "What changed" at the top when updating existing wiki pages.
