Markdown.Rule Skill

Concise reference for markdownlint rules and writing expectations.

Key rules:

- Line length: 100 chars recommended; allow exceptions for code blocks and tables.
- Heading increments: Do not skip heading levels (H1 → H2 → H3...).
- Use fenced code blocks with language identifiers.
- Use consistent list markers (prefer `-` for unordered lists).

Writing profile:

- Use active voice and concise sentences.
- Keep paragraphs short (≤3 sentences) where possible.
